---

## 📐 전체 시스템 구조

### 현재 (Phase 0 - 완료)

| 모듈 | 상태 |
|------|------|
| 📅 캘린더 (주간 일정) | ✅ 완료 |
| ✅ Task List (할일 관리) | ✅ 완료 |
| 📝 Keep (메모) | ✅ 완료 |
| ⭐ 즐겨찾기 | ✅ 완료 |

### 확장 계획 (Phase 1~3)

| Phase | 모듈 | 설명 |
|-------|------|------|
| Phase 1 | 👥 팀원 인사정보 보드 | 팀원 기본 정보 관리 (기초 데이터) |
| Phase 2 | 🧳 출장현황 보드 | 출장 일정 및 이력 관리 |

## 📊 데이터 구조 설계

### 1. TeamMember (팀원 기본 정보) - **핵심 마스터 데이터**

| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| id | string | 고유 ID | "member-001" |
| name | string | 이름 | "김철수" |
| position | string | 직급 | "책임" |
| role | string | 직무/역할 | "SW개발" |
| department | string | 소속 | "기술센터" |
| email | string | 이메일 | "cs.kim@company.com" |
| phone | string | 연락처 | "010-1234-5678" |
| extensionNumber | string | 내선번호 | "3456" |
| joinDate | string | 입사일 | "2020-03-02" |
| birthDate | string | 생년월일 | "1990-05-15" |
| profileImage | string | 프로필 사진 URL | (선택사항) |
| status | string | 상태 | "재직" / "휴직" / "퇴직" |
| notes | string | 비고 | 자유 메모 |

### 2. BusinessTrip (출장)

| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| id | string | 고유 ID | "trip-001" |
| memberId | string | 팀원 ID (연결) | "member-001" |
| destination | string | 출장지 | "부산 해운대" |
| purpose | string | 출장 목적 | "고객사 미팅" |
| startDate | string | 시작일 | "2026-02-15" |
| endDate | string | 종료일 | "2026-02-16" |
| status | string | 상태 | "예정" / "진행중" / "완료" / "취소" |
| transportation | string | 교통수단 | "KTX" / "항공" / "차량" |
| accommodation | string | 숙소 | "해운대 호텔" |
| notes | string | 비고 | 세부사항 |

### 3. Attendance (근태)

| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| id | string | 고유 ID | "att-001" |
| memberId | string | 팀원 ID (연결) | "member-001" |
| date | string | 날짜 | "2026-02-10" |
| endDate | string | 종료일 (연차 등) | "2026-02-11" |
| type | string | 유형 | 아래 표 참고 |
| halfDayType | string | 반차 구분 | "AM" / "PM" |
| reason | string | 사유 | "개인 사유" |
| status | string | 상태 | "승인" / "대기" / "반려" |

**근태 유형 (type):**

| 코드 | 설명 |
|------|------|
| annual | 연차 |
| half | 반차 (오전/오후) |
| sick | 병가 |
| remote | 재택근무 |
| education | 교육 |
| special | 경조사 |
| overtime | 초과근무 |

## 🔗 데이터 연결 구조

```
┌─────────────────────────────────────────────┐
│          TeamMember (팀원 마스터)              │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐    │
│  │김철수 │  │박영희 │  │이준호 │  │최미래 │    │
│  └──┬───┘  └──┬───┘  └──┬───┘  └──┬───┘    │
└─────┼─────────┼─────────┼─────────┼─────────┘
      │         │         │         │
   ┌──┴──┐   ┌──┴──┐   ┌──┴──┐   ┌──┴──┐
   │출장  │   │근태  │   │할일  │   │일정  │
   │이력  │   │현황  │   │담당  │   │주관  │
   └─────┘   └─────┘   └─────┘   └─────┘
```

**핵심: `memberId`로 모든 데이터가 팀원에게 연결됩니다.**

- 출장 → `memberId`로 누구의 출장인지 연결
- 근태 → `memberId`로 누구의 근태인지 연결
- 기존 Task의 `assignee` → `memberId`로 연결 가능
- 기존 Task의 `organizer` → `memberId`로 연결 가능

## 🖥️ 화면 구성 계획

### Phase 1: 팀원 인사정보 보드

| 구성요소 | 설명 |
|----------|------|
| 팀원 카드 리스트 | 이름, 직급, 연락처 카드 형태 |
| 팀원 상세 모달 | 클릭 시 상세 정보 표시 |
| 팀원 추가/수정 폼 | 인사정보 입력 양식 |
| 검색/필터 | 이름, 직급, 소속 검색 |

### Phase 2: 출장현황 보드

| 구성요소 | 설명 |
|----------|------|
| 출장 캘린더 뷰 | 월별 출장 일정 표시 |
| 출장 리스트 뷰 | 팀원별 출장 목록 |
| 출장 등록 모달 | 출장 정보 입력 |
| 출장 현황 요약 | 이번 주/월 출장자 표시 |

### Phase 2: 근태현황 보드

| 구성요소 | 설명 |
|----------|------|
| 근태 캘린더 뷰 | 월별 근태 현황 표시 |
| 근태 리스트 뷰 | 팀원별 연차 사용 현황 |
| 근태 등록 모달 | 연차/반차/재택 등록 |
| 연차 잔여 현황 | 팀원별 남은 연차 표시 |

### Phase 3: 통합 대시보드

| 영역 | 내용 |
|------|------|
| 📅 이번 주 일정 | 캘린더에서 이번 주 주요 일정 |
| ✅ 오늘 할일 | 마감 임박 Task Top 5 |
| 🧳 출장 현황 | 현재 출장자 + 예정 출장 |
| 📊 근태 현황 | 오늘 부재자 (연차/반차/재택) |
| 👥 팀원 상태 | 전체 팀원 현재 상태 한눈에 |
